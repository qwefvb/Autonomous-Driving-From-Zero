# 7.2 科研实践心得

作为一名研一学生，在自动驾驶领域的探索过程中，积累一些高效的科研习惯远比单纯的代码量重要。

## 1. 论文阅读方法论

自动驾驶领域的论文数量庞大，必须学会“精读”与“泛读”结合：
- **第一遍 (Fast Scan)**：阅读 Abstract, Introduction, 图表和结论。判断该方法是否值得深入研究。
- **第二遍 (Deep Dive)**：仔细研读 Method 部分，理解其数学推导和逻辑链条。
- **第三遍 (Implementation)**：对着官方开源代码看论文，理解核心算子（如 Cross-Attention, Deformable Detr）在代码中是如何实现的。

## 2. 实验复现与消融实验

- **严谨的基准 (Baseline)**：在做任何创新前，先确保你能完整复现官方给出的性能指标。
- **消融实验 (Ablation Study)**：清楚每一个模块、每一组超参数对结果的影响。不要一次性改变多个变量。
- **实验记录**：建议使用 **WandB (Weights & Biases)** 记录每一轮训练的曲线和超参数。

## 3. 关注社区动态

- **Arxiv Daily**：关注每天最新的 CS.CV, CS.RO 论文。
- **GitHub Trending**：关注自动驾驶相关的明星项目。
- **知乎/公众号**：许多优秀的博主会对顶会论文（CVPR, ICCV, ECCV, NeurIPS）进行深度解读。

## 4. 心态建设

自动驾驶是一个竞争极其激烈的赛道。保持好奇心，不要因为一时的实验结果不好而气馁。多与同行交流，往往能碰撞出不一样的火花。
