# 2.1.3 2D 图像分割

2D 图像分割是自动驾驶视觉感知中最“细粒度”的任务之一。它不再只回答“图里有什么目标、目标大致在哪里”，而是进一步回答：

- 图像中的每一个像素分别属于什么语义类别。
- 某个目标的精确轮廓到底延伸到哪里。
- 场景中哪些区域是道路、哪些是人行道、哪些是车辆、哪些只是背景结构。

**它通常输出像素级标签图、实例掩码 `mask`，或者统一的全景分割结果。相比 2D 检测只给出边界框，分割提供的是更完整的场景结构信息，因此常被用于可行驶区域识别、动态障碍物轮廓提取、道路结构理解，以及后续 BEV、Occupancy、在线地图等任务的前置语义信息。**

> [!TIP]
> **可以把 2D 检测理解为“把关键目标圈出来”，而把 2D 分割理解为“沿着轮廓把目标描清楚，并给整张图逐像素上色”。检测更像粗定位，分割更像精细理解。**

---

## 1. 2D 分割在自动驾驶中的职责

在自动驾驶里，分割最常承担的不是“替代检测”，而是补上检测做不到的那部分细节理解。

它的典型职责包括：

- **可行驶区域识别**：区分道路、路肩、隔离带、人行道、绿化带。
- **道路结构理解**：识别车道区域、路口开口、停止线附近区域、路缘边界。
- **动态障碍物精细轮廓提取**：给出车辆、行人、骑行者的像素级边界。
- **静态语义地图补充**：建筑、天空、植被、护栏、墙体等背景元素分类。
- **下游任务辅助监督**：为 BEV 感知、占据网络、在线建图提供更强的图像语义先验。

它的典型价值包括：

- 帮助系统更准确地判断“哪里能走，哪里不能走”。
- 帮助目标边界贴合真实轮廓，而不是只给一个矩形框。
- 为细粒度规划和低速精细操作提供更可信的空间边界。
- 为多任务学习提供统一的像素级监督信号。

但它也有天然边界：

- 只是在图像平面上理解语义，不直接给出真实三维距离。
- 对边界标注质量、类别定义和场景长尾比较敏感。
- 单帧分割结果通常没有稳定时序身份，需要跟踪或时序模块补充。
- 高分辨率分割计算量较大，部署时容易受到延迟和带宽限制。

## 2. 基础概念与分割器工作流

### 2.1 一个分割器到底在做什么

一个完整的 2D 分割器，通常同时在做 4 件事：

1. **特征提取**：从原图中提取语义特征。
2. **多尺度上下文融合**：同时兼顾局部边界和全局语义。
3. **像素级预测**：为每个像素输出类别，或输出像素到实例的归属关系。
4. **结果重建与后处理**：将低分辨率特征恢复到更精细的空间结果。

下面这条流程覆盖了大多数分割模型：

```mermaid
graph LR
    A[输入图像] --> B[Backbone 提取特征]
    B --> C[多尺度上下文融合]
    C --> D[Decoder 上采样与细节恢复]
    D --> E[像素分类 / Mask 预测]
    E --> F[实例分组或全景融合]
    F --> G[最终分割结果]
```

和图像分类相比，分割不只是判断“这张图里有没有车”；和检测相比，分割也不只是回答“车大概在哪个框里”。它要进一步回答：

- 这辆车的轮廓到底到哪里结束。
- 路和人行道的分界线在哪里。
- 远处那个只有几像素宽的交通参与者是否还能被完整保留下来。

因此，分割天然要同时处理三件事：

- **语义分类（Semantic Classification）**：每个像素属于哪一类。
- **空间定位（Spatial Localization）**：目标边界是否贴合真实轮廓。
- **结构一致性（Structural Consistency）**：同一物体内部是否连贯、不同类别边界是否清晰。

### 2.2 语义分割、实例分割与全景分割

2D 图像分割通常分成三类任务：

- **语义分割（Semantic Segmentation）**：
  每个像素只预测语义类别，不区分同类目标之间的个体身份。
- **实例分割（Instance Segmentation）**：
  每个像素不仅有类别，还要归属到某一个具体实例。
- **全景分割（Panoptic Segmentation）**：
  把语义分割和实例分割统一起来，既描述背景 `stuff`，也区分前景 `thing`。

如果用输出形式表示：

- 语义分割更像 `class(x, y)`。
- 实例分割更像 `(class(x, y), instance_id(x, y))`。
- 全景分割则要求整张图中的每个像素都被唯一解释。

这里要特别区分两个概念：

- **Thing 类别**：可数、可区分实例的目标，如车辆、行人、骑行者。
- **Stuff 类别**：不可数、通常更连续的背景区域，如道路、天空、植被、建筑立面。

自动驾驶里两者都重要：

- `thing` 决定系统要不要关注某个具体参与者。
- `stuff` 决定系统对整体道路结构的理解是否完整。

> [!TIP]
> **只做语义分割时，两辆并排的车可能在结果图里被涂成同一片 `car` 区域；只有实例分割或全景分割，才会明确“这是两辆不同的车”。**

### 2.3 为什么上采样、跳连和上下文建模如此关键

分割和检测都依赖卷积特征，但分割对空间分辨率更敏感。

原因很直接：

- 主干网络连续下采样后，语义更强了，但像素位置变粗了。
- 如果没有有效的恢复机制，边界会模糊，小目标会消失，细长结构会断裂。

因此，分割模型几乎都会围绕三类设计展开：

- **上采样（Upsampling）**：
  把低分辨率特征恢复到更高分辨率。
- **跳跃连接（Skip Connection）**：
  把浅层高分辨率细节与深层高语义特征融合。
- **上下文建模（Context Modeling）**：
  让模型同时看见局部边界和更大范围的场景语义。

在自动驾驶里，如果没有这些设计，模型很容易出现：

- 路缘和人行道边界糊在一起。
- 远处行人、骑行者被背景吞没。
- 电线杆、护栏、路沿等细长结构断裂。
- 路口大范围区域理解不稳定。

### 2.4 分割常见损失

最基础的语义分割损失通常来自像素级分类，例如交叉熵损失。对一张图像中的像素集合 $\Omega$，可以写成：

$$
L_{ce} = - \sum_{p \in \Omega} \sum_{k=1}^{K} y_{p,k} \log \hat{y}_{p,k}
$$

其中：

- $p$ 表示像素位置。
- $K$ 表示类别数。
- $y_{p,k}$ 表示像素 $p$ 的真实 one-hot 标签。
- $\hat{y}_{p,k}$ 表示模型对类别 $k$ 的预测概率。

但仅靠像素级交叉熵往往不够，因为自动驾驶里常见：

- 背景像素远多于关键前景像素。
- 细边界类在总体像素里占比很低。
- 大面积道路类可能“掩盖”关键小类的错误。

因此很多方法还会加入：

- Dice Loss / IoU Loss：更关注区域重合。
- Boundary Loss：更关注边界质量。
- Focal Loss：缓解类别不平衡。

## 3. 方法演进：

分割方法的发展，并不只是“网络越来越深”，而是围绕几个核心矛盾持续演化：

- 如何在保持全局语义的同时保住像素边界细节。
- 如何兼顾大范围背景区域与小而细的前景目标。
- 如何让实例级目标与背景语义在一个统一框架下建模。
- 如何在精度、分辨率、实时性之间取得工程平衡。

早期图像分割更多依赖：

- 手工纹理、颜色和边缘特征。
- 图割（Graph Cut）、CRF、超像素等传统方法。
- 先分区域、再做规则合并的处理流程。

这些方法的问题在于：

- 特征表达能力有限。
- 对复杂光照、尺度变化和遮挡适应性差。
- 很难在大规模自动驾驶场景上稳定泛化。

深度学习带来的关键变化是：**让网络直接学习“从图像到像素标签”的端到端映射。**

## 4. 语义分割：从 FCN 到上下文建模

### 4.1 FCN：把分类网络改造成像素预测网络

FCN（Fully Convolutional Network）通常被视为现代语义分割的起点。

它最重要的变化是：

- 去掉传统分类网络末端的全连接层。
- 保留卷积结构，让网络输出空间特征图。
- 再通过上采样把低分辨率预测恢复到像素级输出。

这件事的意义非常大，因为它第一次系统地证明：

**语义分割可以被统一地写成端到端的像素级预测任务。**

FCN 还引入了浅层与深层特征融合的思路，这为后来的编码器-解码器结构和跳连设计打下了基础。

> [!TIP]
> **论文链接**：[Fully Convolutional Networks for Semantic Segmentation](https://arxiv.org/abs/1411.4038)

### 4.2 编码器-解码器：U-Net 一类结构为什么有效？

FCN 之后，一个很自然的问题出现了：

> 深层特征语义强，但空间分辨率太低，怎样把边界细节找回来？

编码器-解码器结构给出的答案是：

- 编码器逐步下采样，提取高语义特征。
- 解码器逐步上采样，恢复空间分辨率。
- 跳跃连接把浅层细节直接送到解码阶段。

这一类结构在医学图像、遥感和自动驾驶里都非常常见，因为它非常符合分割的核心需求：

- 深层负责“看懂是什么”。
- 浅层负责“保住边界在哪里”。

`U-Net` 虽然不是自动驾驶语义分割的唯一主线，但它代表了一个重要范式：**恢复分辨率不能只靠粗暴插值，还要把浅层细节重新接回来。**

> [!TIP]
> **论文链接**：[U-Net: Convolutional Networks for Biomedical Image Segmentation](https://arxiv.org/abs/1505.04597)

### 4.3 PSPNet 与 DeepLab：为什么上下文建模变成核心

仅仅恢复分辨率还不够，因为很多像素是否属于某一类，往往取决于更大的上下文范围。

例如：

- 一片灰色区域到底是路面、阴影还是建筑墙体？
- 一个狭长的白色区域到底是车道线还是护栏高光？
- 远处那一小块黑色区域到底是轮胎、井盖还是噪声？

这就是为什么分割方法后来越来越强调 **上下文建模**。

`PSPNet` 的核心思路是：

- 通过金字塔池化聚合不同尺度的全局上下文。
- 让模型同时感知局部纹理和更大范围的场景结构。

而 `DeepLab` 系列则把另一条路线做得非常成熟：

- 用空洞卷积 `dilated convolution` 扩大感受野。
- 在不显著降低分辨率的情况下获取更大的上下文。
- 用 `ASPP (Atrous Spatial Pyramid Pooling)` 聚合多尺度语义。

这类方法之所以在自动驾驶语义分割中长期重要，本质上是因为道路场景既需要：

- 大范围全局理解：知道这里是十字路口、主路还是匝道。
- 又需要局部边界细节：分清路缘、道路边界、可行驶区域边缘。

> [!TIP]
> **论文链接**：
> - [Pyramid Scene Parsing Network](https://arxiv.org/abs/1612.01105)
> - [Rethinking Atrous Convolution for Semantic Image Segmentation](https://arxiv.org/abs/1706.05587)

### 4.4 实时语义分割：为什么自动驾驶特别重视速度

自动驾驶不会只关心离线指标，还必须关心：

- 输入分辨率有多大。
- 单帧推理延迟有多高。
- 模型是否能稳定运行在车端硬件上。

这就推动了一条非常重要的分支：**实时语义分割。**

这类方法的核心问题不是“能不能分得最细”，而是：

> 在有限算力下，怎样保住最关键的语义质量？

典型思路包括：

- 使用轻量级主干网络。
- 将高分辨率细节分支与低分辨率语义分支解耦。
- 用快速特征融合代替过重的解码器。

例如：

- `ENet` 强调轻量和速度，是早期实时分割的重要代表。
- `BiSeNet` 用双分支结构同时保留空间细节和语义上下文。
- `DDRNet` 在高分辨率分支和低分辨率分支之间做双向信息交换，在速度和精度之间取得了不错平衡。

> [!TIP]
> **论文链接**：
> - [ENet: A Deep Neural Network Architecture for Real-Time Semantic Segmentation](https://arxiv.org/abs/1606.02147)
> - [BiSeNet: Bilateral Segmentation Network for Real-time Semantic Segmentation](https://arxiv.org/abs/1808.00897)
> - [Deep Dual-resolution Networks for Real-time and Accurate Semantic Segmentation of Road Scenes](https://arxiv.org/abs/2101.06085)

### 4.5 语义分割的优缺点

**优点**

- 能为整张图提供连续、完整的场景语义。
- 对道路、天空、建筑、植被等背景结构理解非常自然。
- 在可行驶区域识别和道路结构理解中非常有价值。

**缺点**

- 不区分同类实例，多个目标容易连成一片。
- 对细长边界、小目标和稀有类别比较敏感。
- 高分辨率部署的计算成本较高。

## 5. 实例分割：不仅要知道是什么，还要知道是谁

### 5.1 为什么语义分割还不够

如果一张图里有三辆并排的车，语义分割可能会正确地把它们都标成 `car`，但它并不知道这是三辆不同的车。

这在自动驾驶里会直接带来限制：

- 无法区分相邻同类目标的个体身份。
- 无法直接把像素级结果送给多目标跟踪。
- 难以在密集场景中稳定估计每个目标的精确轮廓。

于是，实例分割的目标变成了：

> 不仅要给像素分类，还要把同类目标拆分成一个个独立实例。

### 5.2 Mask R-CNN：先检测，再为每个目标生成 Mask

`Mask R-CNN` 是实例分割历史上的标志性方法。

它延续了 Faster R-CNN 的两阶段思路：

1. 先用检测器提出候选目标区域。
2. 对每个 RoI 进行分类和边框回归。
3. 额外增加一个并行的 mask 分支，为每个目标预测像素级掩码。

它的重要意义在于：

- 把目标检测与像素级分割统一在一个框架内。
- 让实例分割第一次成为成熟、强大、易理解的标准方案。
- 在自动驾驶、通用视觉和工业落地中都产生了深远影响。

从思想上看，`Mask R-CNN` 很符合工程直觉：

- 先找“这个目标大概在哪”。
- 再在局部区域里把轮廓精细切出来。

这种做法在目标边界较清晰、实例数量适中时通常很有效。

> [!TIP]
> **论文链接**：[Mask R-CNN](https://arxiv.org/abs/1703.06870)

### 5.3 从检测驱动到位置驱动：SOLO / SOLOv2 一类方法

两阶段实例分割虽然强，但也有明显代价：

- 依赖检测框质量。
- 流程更复杂。
- 对密集场景和实时部署不一定最友好。

因此，后续很多工作开始探索：

> 能不能不先显式检测 RoI，而是更直接地按位置或实例来生成 mask？

`SOLO` / `SOLOv2` 代表了一条很重要的思路：

- 不再严格依赖“先框后分”的两阶段范式。
- 直接把实例分割建模为按网格或位置预测实例掩码。
- 让同一位置负责某个实例的分割结果。

它的意义在于：

- 降低了对检测框质量的依赖。
- 为更直接、更统一的实例建模打开了空间。

> [!TIP]
> **论文链接**：[SOLOv2: Dynamic and Fast Instance Segmentation](https://arxiv.org/abs/2003.10152)

### 5.4 实例分割在自动驾驶中的优缺点

**优点**

- 能把相邻同类目标明确分开。
- 对车辆、行人、骑行者等动态目标边界刻画更准确。
- 更适合与跟踪、行为预测等实例级下游任务衔接。

**缺点**

- 对遮挡、重叠和远距离小目标依然很敏感。
- 标注成本更高，训练更复杂。
- 相比纯语义分割，部署开销通常更大。

## 6. 全景分割：把 thing 和 stuff 统一起来

### 6.1 为什么还需要全景分割

语义分割擅长理解背景结构，实例分割擅长拆分前景个体，但真实驾驶场景并不会天然把两类问题分开。

系统真正想知道的是：

- 哪些像素属于道路、天空、建筑这类连续背景。
- 哪些像素属于具体的某一辆车、某一个行人。
- 整张图中的每个像素能不能被唯一、完整、无冲突地解释。

这正是全景分割的目标。

它要求：

- 对 `stuff` 类别做语义分割。
- 对 `thing` 类别做实例分割。
- 将两者融合成一张全局一致的结果图。

从自动驾驶角度看，全景分割比“单独做语义 + 单独做实例”更接近真实需求，因为它更像一种完整的场景解释结果。

### 6.2 全景分割的意义与工程价值

全景分割最重要的价值不是单纯提高某个指标，而是统一建模视角：

- 对规划和场景理解而言，整张图中每个像素都有语义归属。
- 对动态目标建模而言，关键交通参与者有明确实例身份。
- 对多任务系统而言，更容易作为统一的视觉中间表征。

在自动驾驶里，这意味着它可以更自然地服务于：

- 场景结构解析
- 语义地图补充
- BEV 语义投影
- 占据建模与自由空间理解

### 6.3 Panoptic Quality 为什么重要

全景分割常用 `Panoptic Quality (PQ)` 评估。它常写成：

$$
PQ = SQ \times RQ
$$

其中：

- $SQ$ 表示 `Segmentation Quality`，反映已匹配实例的分割质量。
- $RQ$ 表示 `Recognition Quality`，反映识别和匹配是否成功。

这类指标的重要性在于：它不只关心“有没有分对类别”，还关心“有没有把实例和区域整体解释清楚”。

---

## 7. Transformer 分割：统一分割范式的新方向

Transformer 进入分割领域后，最重要的变化不是简单换一个主干网络，而是进一步推动了：

- 长程依赖建模
- 多尺度特征统一融合
- 不同分割任务之间的统一表达

### 7.1 SegFormer：轻量而强的语义分割代表

`SegFormer` 是近几年语义分割里非常有代表性的模型。

它的特点包括：

- 使用层次化 Transformer 编码器提取多尺度特征。
- 解码头相对轻量，但融合方式高效。
- 在精度和工程落地之间取得了很好的平衡。

它的重要意义在于：证明了 Transformer 并不一定意味着极重的结构，合理设计后也能成为很实用的分割主干。

> [!TIP]
> **论文链接**：[SegFormer: Simple and Efficient Design for Semantic Segmentation with Transformers](https://arxiv.org/abs/2105.15203)

### 7.2 Mask2Former：从“任务专用头”走向统一 Mask 预测

`Mask2Former` 代表了另一条更具范式意义的路线：

- 用统一的 mask 分类框架处理不同分割任务。
- 把语义分割、实例分割和全景分割尽量放进同一建模语言里。
- 借助 Transformer 的查询机制做更灵活的区域级预测。

它的重要价值不只是性能强，还在于它体现了一种趋势：

> 分割不一定要被拆成许多彼此割裂的任务头，而可以在统一 mask 预测框架下表达。

这对自动驾驶多任务感知非常有启发，因为车端系统本来就不希望维护过多风格完全不同的任务分支。

> [!TIP]
> **论文链接**：[Masked-attention Mask Transformer for Universal Image Segmentation](https://arxiv.org/abs/2112.01527)

### 7.3 Transformer 分割的优缺点

**优点**

- 全局关系建模能力强。
- 统一不同分割任务的潜力更大。
- 更适合向通用视觉表征和多任务学习扩展。

**缺点**

- 训练资源和调参复杂度通常更高。
- 高分辨率输入下显存与延迟压力仍然明显。
- 真正落地时仍要权衡车端算力与模型收益。

## 8. 自动驾驶场景下的典型难点

### 8.1 边界细节、小目标与遮挡

自动驾驶分割最大的难点之一，是许多“真正重要”的目标恰恰非常难分：

- 远处行人和骑行者只占极少像素。
- 路缘、护栏、交通锥属于细长或高频结构。
- 相互遮挡的车辆和行人边界非常复杂。

这会导致一些典型问题：

- 目标轮廓被过度平滑。
- 相邻实例黏连在一起。
- 细小目标直接消失在背景里。

### 8.2 光照、天气与路面材质变化

真实道路场景里，图像分割并不是在“干净数据集”上进行的。

模型必须面对：

- 夜晚低照度
- 雨雪雾天气
- 逆光、眩光和车灯反射
- 路面积水、阴影、沥青老化和施工涂改

这些因素都会直接破坏像素外观一致性，让模型更难仅靠局部纹理做判断。

### 8.3 类别长尾与标注边界不确定性

分割比检测更依赖标注质量，因为边界本身就是监督目标的一部分。

现实中常见问题包括：

- 少数关键类样本极少，例如施工锥桶、拖挂车、异常障碍物。
- 标注人员对边界位置可能存在主观差异。
- 类别定义是否合并、是否细分，会显著影响训练目标。

这也是为什么分割项目往往不只是“换个模型”，而是还要反复打磨：

- 标签体系
- 数据采样策略
- 边界质量控制
- 长尾类别权重设计

## 9. 训练评估、模型选型与工程判断

### 9.1 Pixel Accuracy 与 mIoU

语义分割最常见的两个指标是 `Pixel Accuracy` 和 `mIoU`。

其中，单类别 IoU 可写作：

$$
\text{IoU}_k = \frac{TP_k}{TP_k + FP_k + FN_k}
$$

进一步地，平均交并比 `mIoU` 写作：

$$
\text{mIoU} = \frac{1}{K}\sum_{k=1}^{K}\frac{TP_k}{TP_k + FP_k + FN_k}
$$

其中：

- $TP_k$ 表示类别 $k$ 的真正例像素数。
- $FP_k$ 表示被错误预测为类别 $k$ 的像素数。
- $FN_k$ 表示真实属于类别 $k$ 但没有预测对的像素数。

`Pixel Accuracy` 容易理解，但它可能被大面积背景类别“冲高”；`mIoU` 更能反映各类别的真实分割质量，因此通常更常用。

不过在自动驾驶里，不能只盯着一个总体 `mIoU`，还要看：

- 关键小类的 IoU 是否过低。
- 细长边界类是否容易断裂。
- 可行驶区域边界是否稳定。
- 不同天气和时段下是否有明显退化。

### 9.2 实例分割的 AP 与 mask AP

实例分割常沿用检测中的 `AP` 评估思想，但关注的是 mask 重合质量而不只是框重合。

因此常会看到：

- `box AP`
- `mask AP`

其中 `mask AP` 更能反映实例轮廓是否被准确切分出来。

在自动驾驶里，这类指标对以下场景特别关键：

- 密集车辆场景
- 人车混行场景
- 遮挡严重场景

因为这些场景里，“有没有检测到”只是第一步，“边界是否分清”同样重要。

### 9.3 全景分割的 PQ、SQ、RQ

相比只看语义或实例，全景分割更常使用：

- `PQ (Panoptic Quality)`
- `SQ (Segmentation Quality)`
- `RQ (Recognition Quality)`

它们的价值在于：

- 同时评估区域质量和识别成功率。
- 更适合统一衡量 `stuff + thing` 的整体结果。

如果一个模型把道路分得很好，却总是把相邻车辆实例粘在一起，那么它的 `PQ` 往往不会好看。

### 9.4 FPS、Latency 与输入分辨率的真实取舍

自动驾驶里的模型选型，从来不只是论文排行榜问题。

工程上还必须同时看：

- `FPS`
- 单帧延迟 `Latency`
- 输入分辨率
- 显存/带宽占用
- 目标部署平台

因为分辨率越高，边界通常越细，但计算成本也越高；模型越强，精度可能越高，但端侧实时性压力也越大。

## 10. 学习路径、资料与实践建议

### 10.1 一个推荐的入门顺序

如果你刚开始系统学习 2D 图像分割，一个比较顺的路径是：

1. 先理解语义分割的基本任务定义与 `mIoU`。
2. 再掌握 `FCN`、`U-Net`、`DeepLab` 这条主线。
3. 接着理解实例分割与 `Mask R-CNN` 的核心思想。
4. 再把全景分割看成 `stuff + thing` 的统一建模问题。
5. 最后进入 `SegFormer`、`Mask2Former` 等更现代的统一范式。

这样学习的好处是：先把“像素分类”学清楚，再逐步过渡到“实例拆分”和“统一场景解释”。

### 10.2 推荐数据集与开源框架

推荐优先了解以下数据集：

- `Cityscapes`：城市街景语义分割的经典基准，类别定义和评测体系都很成熟。
- `BDD100K`：覆盖天气、时段和交通场景变化更丰富，贴近真实道路长尾。
- `Mapillary Vistas`：类别更细、场景更多样，对泛化能力要求更高。

推荐优先了解以下开源框架：

- `MMSegmentation`：适合系统学习语义分割模型与训练配置。
- `MMDetection`：适合理解实例分割与全景分割相关实现。
- `Detectron2`：也是实例分割、全景分割领域非常重要的工程框架。

> [!TIP]
> **资料链接**：
> - [Cityscapes Dataset](https://www.cityscapes-dataset.com/)
> - [BDD100K](https://bdd-data.berkeley.edu/)
> - [Mapillary Vistas](https://www.mapillary.com/dataset/vistas)
> - [MMSegmentation](https://github.com/open-mmlab/mmsegmentation)
> - [MMDetection](https://github.com/open-mmlab/mmdetection)

### 10.3 小结与实践建议

如果只用一句话概括 2D 图像分割在自动驾驶中的价值，那就是：

**它负责把“这张图里大概有什么”推进成“这张图里每一块区域到底是什么、边界到底在哪里”。**

分割并不能独立解决所有感知问题，但它提供了 2D 视觉里最细粒度、最接近“完整场景解释”的语义信息，这也是它在自动驾驶感知系统中长期重要的原因。
