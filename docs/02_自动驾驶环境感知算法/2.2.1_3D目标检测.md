# 2.2.1 3D 目标检测 (单目/Lidar/多模态)

3D 目标检测是自动驾驶感知的核心任务，它不仅要识别图像中物体的类别（分类），还要确定其在 3D 空间中的具体位置、尺寸和朝向（$x, y, z, w, h, l, \theta$）。

---

## 1. 为什么需要从 2D 转向 3D？

2D 检测虽然成熟，但无法提供深度信息。在自动驾驶中，如果只知道前方有一个行人（2D 框），而不知道他距离本车 5 米还是 50 米，规划算法将无法做出刹车或绕行的决策。

**3D 检测的目标是获取物体的 7 个自由度：**
- **位置**：$x, y, z$ (3D 中心坐标)
- **尺寸**：$w, h, l$ (宽、高、长)
- **偏航角**：$\theta$ (朝向)

---

## 2. 单目 3D 目标检测 (Monocular 3D)

单目检测仅依赖一个摄像头，通过 2D 图像预测 3D 信息。由于缺乏直接的深度感知，这是一个数学上的**不适定问题 (Ill-posed Problem)**。

### 2.1 核心挑战
- **深度缺失**：同一个 2D 框可能对应无限个 3D 框（近大远小）。
- **几何约束**：通常需要利用地平面假设、物体的先验尺寸（如轿车平均长 4.5m）来辅助估计。

### 2.2 经典算法
- **SMOKE**：舍弃 2D 候选框，直接预测 3D 中心投影点。
- **MonoCon**：利用 2D 关键点约束来学习 3D 几何特征。
- **PGD**：引入几何深度分解，提升深度估计的准确性。

---

## 3. Lidar 3D 目标检测 (Point Cloud)

激光雷达（Lidar）提供精确的深度和空间拓扑，是 3D 感知的绝对主力。

### 3.1 点云表征方式
由于点云是无序且稀疏的，神经网络不能直接处理原始点。
1.  **Voxel (体素)**：将空间切分为小方块（3D 栅格），代表作：**SECOND**。
2.  **Pillar (柱体)**：体素在 Z 轴上的简化，将点云展平为伪图像（Pseudo Image），代表作：**PointPillars**（量产最常用，速度极快）。
3.  **Point-based**：直接对原始点进行采样和聚合，代表作：**PointRCNN**。

### 3.2 经典模型
- **CenterPoint**：目前主流的 Anchor-free 架构，通过预测中心点热力图来回归 3D 框。
- **PV-RCNN**：结合了 Voxel 和 Point 的优点，精度极高。

---

## 4. 多模态 3D 目标检测 (Fusion)

融合“精确但稀疏”的 Lidar 和“细致但无深度”的图像，实现 1+1 > 2 的效果。

### 4.1 融合策略
- **早融合 (Early Fusion)**：将点云投影到图像，拼接到特征图上。
- **深层特征融合 (Feature Fusion)**：在 BEV 空间或特征层进行对齐。
- **代表工作**：**BEVFusion (MIT/阿里)**。它通过将图像特征和点云特征统一变换到 BEV 空间进行高效融合，是目前多模态感知的标杆。

---

## 5. 算法对比与总结

| 感知方案 | 优势 | 劣势 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **单目 3D** | 成本极低、部署方便 | 深度精度差、可靠性低 | 辅助感知、视觉冗余 |
| **Lidar 3D** | 深度极其精确、全天候 | 传感器昂贵、稀疏 | 核心定位与障碍物感知 |
| **多模态** | 精度最高、鲁棒性强 | 算力消耗大、对齐困难 | 高阶自动驾驶 (L3/L4) |
