# 2.2 3D感知与建图

3D 感知是自动驾驶从“看懂图片”到“理解空间”的跃迁。它是下游规划算法能够直接使用的输入。

## 1. 3D 目标检测

### 1.1 单目 3D 检测 (Monocular)
仅使用一个摄像头。由于缺乏深度信息，这是一个极具挑战性的“病态问题”。
- **经典算法**：SMOKE, MonoCon, PGD。

### 1.2 环视图 3D 检测 (Surround-View / BEV)
利用车身周围的多个摄像头（通常是 6 个）合成鸟瞰图 (Bird's Eye View)。这是目前量产自动驾驶最主流的方案。
- **代表工作**：
  - **LSS (Lift, Splat, Shoot)**：BEV 感知的鼻祖，将图像特征“举”到 3D 空间。
  - **BEVFormer**：利用时序信息和 Transformer 构建高性能 BEV 特征。
  - **PETR**：引入 3D Position Embedding 的位置编码范式。

### 1.3 Lidar 3D 检测
利用激光雷达点云，具有天然的精确深度和空间结构。
- **经典算法**：PointPillars (速度快), SECOND, CenterPoint。

### 1.4 多模态 3D 检测 (Multi-modal)
融合图像和点云。
- **代表工作**：BEVFusion (MIT/阿里)。

---

## 2. 在线建图算法 (Online Mapping)

不再依赖高精地图 (HD Map)，而是实时从传感器输入中“绘制”出道路拓扑。
- **代表工作**：**MapTR**, HDMapNet。

---

## 3. 3D Occupancy (占用栅格)

将空间切分为小方块 (Voxel)，预测每个方块是否被占据。这能很好地处理“通用障碍物”（异形车、掉落物等）。
- **代表工作**：**TPVFormer**, **FB-OCC**。
