# 2.1.1 2D 目标检测

目标检测是感知的核心任务，负责识别并定位图像中的物体。

## 1. 两阶段目标检测框架

两阶段检测器（Two-Stage Detector）的核心思想是将目标检测任务解耦为两个串行的子任务：**候选区域生成（Region Proposal）**与**区域特征分类与精修（Classification & Refinement）**。

### 1.1 基本流程与原理

之所以被称为“两阶段”，是因为这类算法在逻辑上分为先后衔接的两个步骤：

1.  **第一阶段（候选框生成）**：从原始图像中搜索出可能包含目标的候选区域（Proposals）。
2.  **第二阶段（识别与回归）**：对每个候选区域提取特征，利用分类器判断所属类别，并利用回归器对位置进行微调。

相比单阶段检测，两阶段框架虽然推理速度较慢，但在定位精度和对小目标的检测能力上通常更具优势。

### 1.2 算法演进：从 R-CNN 到 Faster R-CNN

>

#### R-CNN (Region-based CNN)

R-CNN 是将深度学习引入目标检测的开山之作，奠定了两阶段（Two-stage）检测的基础框架。

- **技术流程**：
  1.  **候选区域提取**：使用 Selective Search 算法在图像中生成约 2000 个类别无关的候选框。
  2.  **特征提取**：将每个候选框强制缩放（Warping）至 $ 227 \times 227 $ 尺寸，送入预训练的 AlexNet 提取 4096 维特征向量。
  3.  **分类与回归**：使用特定类别的线性 SVM 进行分类，并利用线性回归器精修边框位置（Bounding Box Regression）。
- **核心瓶颈**：
  - **计算冗余**：2000 个候选框需分别运行 CNN，提取一次特征需数秒。
  - **训练复杂**：流程分多个阶段进行（微调 CNN $\rightarrow$ 训练 SVM $\rightarrow$ 训练回归器），无法端到端训练。

> [!TIP]
> **论文链接**：[Rich feature hierarchies for accurate object detection and semantic segmentation (CVPR 2014)](https://arxiv.org/abs/1311.2524)

#### SPP-net (Spatial Pyramid Pooling Network)

SPP-net 针对 R-CNN 的计算冗余和输入尺寸限制（强制缩放导致的物体变形）提出了关键改进。

- **关键技术**：
  - **特征图共享**：整张图仅需经过一次卷积层提取全局特征图。各候选框直接在特征图上进行区域映射，避免了重复计算。
  - **空间金字塔池化（SPP Layer）**：引入多尺度池化网格（如 $ 1 \times 1, 2 \times 2, 4 \times 4 $），将任意尺寸的候选区域特征块转化为固定长度的输出向量，解决了 R-CNN 必须输入固定尺寸图像的问题。
- **局限性**：虽然推理速度提升显著，但训练依然是多阶段的，且 SPP 层之前的卷积层在反向传播时无法更新。

> [!TIP]
> **论文链接**：[Spatial Pyramid Pooling in Deep Convolutional Networks for Visual Recognition (ECCV 2014)](https://arxiv.org/abs/1406.4729)

#### Fast R-CNN

Fast R-CNN 融合了 R-CNN 和 SPP-net 的优点，并实现了训练过程的“一体化”。

- **核心改进**：
  - **RoI Pooling**：一种简化版的单层 SPP。它将不同大小的特征块对齐到固定尺寸（通常为 $ 7 \times 7 $），使得后续全连接层可以正常工作。
  - **多任务损失（Multi-task Loss）**：将分类（Softmax）与边框回归（Smooth L1 Loss）整合进同一个网络进行同步训练。
- **优势**：实现了接近端到端的训练（除候选框生成外），其训练速度比 R-CNN 快 9 倍，推理速度大幅提升。

> [!TIP]
> **论文链接**：[Fast R-CNN (ICCV 2015)](https://arxiv.org/abs/1504.08083)

#### Faster R-CNN

Faster R-CNN 是两阶段检测器的里程碑，它真正实现了“全卷积”目标检测并解决了速度瓶颈。

- **核心突破：RPN (Region Proposal Network)**：
  - 使用神经网络替代了传统的 Selective Search 算法。RPN 通过与检测网络共享卷积特征，几乎以“零成本”生成高质量候选框。
  - **Anchor 机制**：在特征图的每个位置预设 9 种不同尺度（Scale）和长宽比（Ratio）的锚框（Anchors），涵盖了不同大小和形状的目标。
- **多任务损失函数**：

$$
L(\{p_{i}\}, \{t_{i}\}) = \frac{1}{N_{cls}} \sum_{i} L_{cls}(p_{i}, p_{i}^{*}) + \lambda \frac{1}{N_{reg}} \sum_{i} p_{i}^{*} L_{reg}(t_{i}, t_{i}^{*})
$$

其中， $ p*{i} $ 为第 $ i $ 个锚框包含物体的概率， $ t*{i} $ 为预测的坐标向量， $ p\_{i}^{\*} $ 为真实标签（Ground Truth）。

> [!TIP]
> **论文链接**：[Faster R-CNN: Towards Real-Time Object Detection with Region Proposal Networks (NIPS 2015)](https://arxiv.org/abs/1506.01497)

## 2. 单阶段目标检测框架

- **YOLO 系列 (One-Stage)**：速度极快，是量产自动驾驶系统的首选。目前已经演进到 YOLOv11。
- **Faster R-CNN (Two-Stage)**：经典的基于候选区域 (Anchor) 的方法。
- **DETR (Transformer-based)**：消除了 NMS（非极大值抑制）等复杂后处理，实现了端到端的检测。

## 3. 基于 Transformer 的目标检测框架

Transformer 架构的引入彻底改变了目标检测的范式，实现了从“手工设计算子”向“端到端集合预测”的转变。

### 3.1 DETR (DEtection TRansformer)

DETR 是将 Transformer 引入目标检测的开山之作，它抛弃了传统的锚框（Anchors）和非极大值抑制（NMS），通过二分图匹配（Bipartite Matching）实现目标定位。

- **核心机制**：利用 CNN 提取图像特征，将其展平后输入 Transformer Encoder；Decoder 接收一组固定的、可学习的 **Object Queries**，通过交叉注意力（Cross-Attention）从特征中提取物体信息。
- **训练损失**：采用全局一致的二分图匹配损失，确保预测值与真值（Ground Truth）的一一对应：

$$
\hat{\sigma} = \arg \min_{\sigma \in \mathfrak{S}_{N}} \sum_{i}^{N} L_{match}(y_{i}, \hat{y}_{\sigma(i)})
$$

- **核心缺陷**：
  1. **收敛极慢**：通常需要 500 个 Epoch 才能达到最佳精度（收敛速度约为 Faster R-CNN 的 1/10）。
  2. **小目标检测差**：受限于全局自注意力的高内存开销，难以处理高分辨率特征图。
  3. **查询语义模糊**：Object Queries 缺乏明确的空间或物理含义。

> [!TIP]
> **论文链接**：[End-to-End Object Detection with Transformers (ECCV 2020)](https://arxiv.org/abs/2005.12872)

### 3.2 技术演进与改进工作

为了克服 DETR 的局限性，学界提出了一系列改进方案，其演进逻辑主要围绕 **“注意力效率”**、**“查询语义化”** 和 **“匹配稳定性”** 展开。

#### Deformable DETR

- **改进点**：引入了 **可变形注意力机制（Deformable Attention）**。它不再对全图进行计算，而是让每个 Query 只关注参考点附近的少量采样点。
- **效果**：极大地加速了收敛过程（训练时间减少约 10 倍），并通过多尺度特征图处理显著提升了对小目标的检测能力。

> [!TIP]
> **论文链接**：[Deformable DETR: Deformable Transformers for End-to-End Object Detection (ICLR 2021)](https://arxiv.org/abs/2010.04159)

#### Anchor DETR

- **改进点**：提出了 **基于锚点的查询设计**。将 Object Queries 显式初始化为图像中的坐标点 $ (x, y) $ 。
- **效果**：为模型提供了明确的空间先验，使得 Decoder 能够更有针对性地从特定区域提取特征。

> [!TIP]
> **论文链接**：[Anchor DETR: Query Design for Transformer-Based Object Detection (AAAI 2022)](https://arxiv.org/abs/2109.07107)

#### DAB-DETR (Dynamic Anchor Box DETR)

- **改进点**：将锚点扩展为 **4D 动态锚框 $ (x, y, w, h) $** 。每一层 Decoder 都会基于前一层的输出对锚框进行位置和尺寸的精修。
- **联系**：它是对 Anchor DETR 的深化，将“检测”建模为一个迭代式的框精炼过程。

> [!TIP]
> **论文链接**：[DAB-DETR: Dynamic Anchor Boxes are Better Queries for DETR (ICLR 2022)](https://arxiv.org/abs/2201.12329)

#### DN-DETR (Denoising DETR)

- **改进点**：引入了 **去噪训练（Denoising Training）**。在训练阶段，将带有随机噪声的真值框作为额外的 Query 输入网络，要求模型将其“还原”回真值。
- **原理**：揭示了 DETR 收敛慢的主因是二分图匹配的不稳定性。去噪任务为模型提供了一个学习匹配的“捷径”，从而大幅缩短了训练周期。

> [!TIP]
> **论文链接**：[DN-DETR: Accelerate DETR Training by Introducing Query DeNoising (CVPR 2022)](https://arxiv.org/abs/2203.01305)

#### DINO (DETR with Improved DeNoising Anchor Boxes)

- **地位**：当前的集大成者，也是首个在性能上全面超越卷积架构（如 ResNet/Swin）的 Transformer 检测器。
- **创新点**：集成了动态锚框和去噪思路，并提出了 **对比去噪（Contrastive DN）**、**混合查询选择**等技术，实现了极高的精度与更快的收敛。

> [!TIP]
> **论文链接**：[DINO: DETR with Improved DeNoising Anchor Boxes for End-to-End Object Detection (ICLR 2023)](https://arxiv.org/abs/2203.03605)

---

### 3.3 总结：从 DETR 到 DINO 的演变主线

1. **查询演进**：抽象 Embedding $\rightarrow$ 2D 空间锚点 $\rightarrow$ 4D 动态锚框。
2. **效率演进**：全局自注意力 $\rightarrow$ 稀疏可变形注意力。
3. **训练演进**：纯匹配驱动 $\rightarrow$ 引入去噪任务辅助匹配。
