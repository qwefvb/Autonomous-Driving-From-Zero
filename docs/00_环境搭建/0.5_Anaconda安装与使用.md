# 0.5 Anaconda 安装与使用全指南

在自动驾驶开发中，我们需要处理海量的依赖库（如 PyTorch, TensorFlow, OpenCV, ROS 等）。不同的项目可能需要完全不同的库版本。**Anaconda** 是解决这一难题的核心工具。

---

## 1. 什么是 Anaconda？为什么要安装它？

### 什么是 Anaconda？

Anaconda 是一个开源的 Python 和 R 语言的发行版本，专为科学计算、数据分析和机器学习设计。它包含了 Conda、Python 以及超过 1500 个常用的科学计算包。

- **Conda**: Anaconda 核心的包管理和虚拟环境管理系统。
- **Miniconda**: Anaconda 的精简版，只包含 Conda 和基本的 Python 环境，适合追求极致空间利用率或在服务器上部署的开发者。

### 为什么需要它？

> [!IMPORTANT]
> 在项目开发中，环境配置往往比代码编写更耗时。Anaconda 的核心价值在于：

1.  **虚拟环境隔离**：你可以为“感知算法”创建一个 Python 3.8 的环境，同时为“规划控制”创建一个 Python 3.10 的环境，互不干扰。
2.  **依赖自动解决**：Conda 拥有强大的依赖解析引擎，能自动帮你下载并安装相互兼容的库版本。
3.  **跨平台一致性**：无论在 Ubuntu 还是 Windows 上，Conda 的操作逻辑完全一致。
4.  **无需 Root 权限**：所有库都安装在用户目录下，避免了污染系统自带的 Python 环境。

## 2. 详细安装步骤（以 Ubuntu 为例）

### 第一步：下载安装包

前往 [Anaconda 官网](https://www.anaconda.com/download) 或 [Miniconda 官网](https://docs.conda.io/en/latest/miniconda.html) 获取下载链接。在终端中使用 `wget` 更快：

```bash
# 下载 Miniconda (推荐：轻量且快速)
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh

# 或者下载完整的 Anaconda (包含大量预装包，约 600MB+)
# wget https://repo.anaconda.com/archive/Anaconda3-2024.02-1-Linux-x86_64.sh
```

### 第二步：执行安装脚本

```bash
bash Miniconda3-latest-Linux-x86_64.sh
```

1.  **阅读协议**：持续按 `Enter` 键直到看到协议末尾。
2.  **接受协议**：输入 `yes` 并回车。
3.  **确认路径**：默认为 `~/miniconda3`，直接按 `Enter` 确认。
4.  **初始化**：当询问 `Do you wish the installer to initialize Miniconda3...` 时，务必输入 **`yes`**。这会自动将 Conda 添加到你的环境变量中。

### 第三步：激活配置

安装完成后，执行以下命令让配置立即生效：

```bash
source ~/.bashrc
```

如果终端提示符前出现了 `(base)` 字样，恭喜你，安装成功！

## 3. Conda 常用指令大全

### 3.1 环境管理 (Environment)

| 功能               | 指令                                            |
| :----------------- | :---------------------------------------------- |
| **创建环境**       | `conda create -n <env_name> python=3.9`         |
| **激活环境**       | `conda activate <env_name>`                     |
| **退出环境**       | `conda deactivate`                              |
| **列出所有环境**   | `conda env list` 或 `conda info --envs`         |
| **删除环境**       | `conda remove -n <env_name> --all`              |
| **克隆环境**       | `conda create -n <new_name> --clone <old_name>` |
| **导出环境配置**   | `conda env export > environment.yml`            |
| **从配置创建环境** | `conda env create -f environment.yml`           |

### 3.2 包管理 (Package)

| 功能             | 指令                           |
| :--------------- | :----------------------------- |
| **安装包**       | `conda install <package_name>` |
| **指定版本安装** | `conda install numpy=1.23.5`   |
| **查看已安装包** | `conda list`                   |
| **搜索包版本**   | `conda search <package_name>`  |
| **卸载包**       | `conda remove <package_name>`  |
| **更新包**       | `conda update <package_name>`  |
| **更新所有包**   | `conda update --all`           |

### 3.3 系统维护 (Maintenance)

- **查看版本**：`conda --version`
- **更新 Conda 自身**：`conda update conda`
- **清理缓存 (释放空间)**：`conda clean --all` (建议定期执行)

## 4. 换源：为什么以及怎么做？

### 为什么要换源？

> [!WARNING]
> 默认的 Conda 源服务器位于海外。在国内直接使用常会导致以下问题：
>
> - 下载速度极慢（几十 KB/s）。
> - 连接超时导致安装中断。
> - 无法获取最新的包。

通过切换到国内的镜像源（如清华大学、中科大、阿里巴巴），下载速度通常能提升 **10-100 倍**。

### 4.1 Conda 换源 (以清华源为例)

执行以下指令直接修改配置文件：

```bash
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/pytorch/
conda config --set show_channel_urls yes
```

- **查看当前源**：`conda config --show channels`
- **恢复默认源**：`conda config --remove-key channels`

### 4.2 Pip 换源 (建议同步操作)

Conda 环境中也经常使用 `pip`。同样建议将 pip 换至国内源：

```bash
# 永久修改
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

## 5. 🛠️ 最佳实践与避坑指南

1.  **严禁在 `base` 环境干活**：`base` 环境仅用于管理其他环境。始终为新项目创建独立环境。
2.  **优先使用 Conda，其次 Pip**：
    - 尽量先尝试 `conda install`。
    - 如果 Conda 找不到包，再使用 `pip install`。
3.  **指定 Python 版本**：创建环境时务必带上 `python=x.x`，否则会默认安装最新版，可能导致兼容性问题。
4.  **环境命名规范**：推荐使用 `项目名_env` 的格式，例如 `lidar_detection_env`。
