# 0.6 远程 SSH 连接与使用 🚀

在项目开发中，由于模型训练和大规模仿真对计算资源（如 GPU）要求极高，我们通常在本地编写代码，而在强大的 **Ubuntu 服务器**上运行程序。**SSH**（Secure Shell）就是实现这一过程的核心桥梁。

---

## 1. 什么是 SSH？有什么用？ 🤔

### 1.1 概念

**SSH (Secure Shell)** 是一种加密的网络传输协议。它允许你在不安全的网络中，为两台电脑之间提供安全的传输服务。

### 1.2 核心用途

- **远程控制**：像坐在服务器面前一样通过命令行操作它。
- **文件传输**：安全地在本地和服务器之间传输代码、数据集或权重文件。
- **远程开发**：利用 VS Code 或 PyCharm 直接修改服务器上的代码并运行。

---

## 2. 基础：建立你的第一次连接 💻

在开始之前，请确保你已知晓服务器的 **IP 地址**、**用户名** 和 **密码**。

### 在 Windows 上

Windows 10/11 已内置 SSH 客户端。

1. 按下 `Win + R`，输入 `cmd` 或 `powershell` 并回车。
2. 输入以下命令（将 `username` 和 `ip` 替换为你的）：
   ```bash
   ssh username@192.168.1.100
   ```
3. 首次连接会提示 `Are you sure you want to continue connecting?`，输入 `yes`。
4. 输入密码（**注意：输入密码时屏幕不会显示任何字符，输完直接回车即可**）。

### 在 macOS 上

1. 打开“终端 (Terminal)”。
2. 输入同样的命令：
   ```bash
   ssh username@192.168.1.100
   ```

## 3. 进阶：配置 SSH 密钥对（免密登录） 🔑

每次输入密码非常麻烦，通过“密钥对”可以实现安全且丝滑的自动登录。

> [!TIP]
> **密钥对原理**：
>
> - **私钥 (Private Key)**：存放在你的本地电脑，绝对不能泄露给别人。
> - **公钥 (Public Key)**：存放在服务器上。
>   登录时，两者匹配成功即可进入。

### 第一步：在本地生成密钥对

在你的本地终端（Windows CMD 或 Mac Terminal）输入：

```bash
ssh-keygen -t rsa -b 4096
```

- 一直按 **回车** 即可（默认不设置密钥密码）。
- 生成成功后，你会在 `~/.ssh/`（即用户目录下的 .ssh 文件夹）看到两个文件：
  - `id_rsa` (私钥)
  - `id_rsa.pub` (公钥)

### 第二步：将公钥上传到服务器

我们需要把本地的 `id_rsa.pub` 的内容告诉服务器。

**方法 A（推荐，极简）：**

```bash
ssh-copy-id username@192.168.1.100
```

输入一次密码后，以后再登录就不再需要密码了。

**方法 B（如果 ssh-copy-id 命令不可用）：**

1. 手动打开本地 `id_rsa.pub` 文件，复制里面的全部内容。
2. SSH 登录服务器，运行：
   ```bash
   mkdir -p ~/.ssh
   echo "刚才复制的内容" >> ~/.ssh/authorized_keys
   chmod 600 ~/.ssh/authorized_keys
   ```

## 4. 在 VS Code 中配置远程开发 🧊

VS Code 是目前最流行的远程开发工具，体验非常接近本地开发。

### 步骤详解：

1. **安装插件**：在左侧插件栏搜索 `Remote - SSH` 并安装。
2. **打开连接面板**：点击左下角的 **蓝色图标 `< >`**，在弹出菜单中选择 `Connect to Host...` -> `Add New SSH Host...`。
3. **输入指令**：
   ```bash
   ssh username@192.168.1.100
   ```
4. **保存配置**：选择默认的配置文件（通常是 `C:\Users\你的名字\.ssh\config`）。
5. **开始连接**：再次点击左下角蓝色图标 -> `Connect to Host` -> 选择刚才添加的 IP。
6. **打开文件夹**：连接成功后，点击左侧“打开文件夹”，你就可以直接浏览服务器上的代码了！

---

## 5. 在 PyCharm 中配置远程开发 🐍

如果你更习惯使用 PyCharm (专业版 Professional)，它对深度学习项目的支持非常强大。

### 步骤详解：

1. **添加 Interpreter**：`File` -> `Settings` -> `Project: xxx` -> `Python Interpreter`。
2. **选择 SSH**：点击右上角 `Add Interpreter` -> `On SSH...`。
3. **配置参数**：
   - `Host`: 服务器 IP
   - `Username`: 用户名
   - 点击 `Next`，如果你配置了密钥，它会自动识别；否则输入密码。
4. **映射目录 (Mappings)**：
   - 告诉 PyCharm 本地代码文件夹对应服务器上的哪个路径。
   - 这样当你保存代码时，PyCharm 会自动将其同步到服务器。
5. **完成**：现在的运行按钮（Run/Debug）将直接在服务器的环境中执行。

---

## 6. 高级技巧：使用 Config 文件简化连接 📝

如果你有多个服务器，记 IP 很痛苦。你可以修改本地的 `~/.ssh/config` 文件：

```text
Host my_server
    HostName 192.168.1.100
    User acheng
    Port 22
    IdentityFile ~/.ssh/id_rsa
```

配置好后，你只需要输入 `ssh my_server` 即可一键登录！

---

> [!IMPORTANT]
> **安全警示**：
> 请妥善保管你的 `id_rsa` 私钥文件。如果服务器暴露在公网，建议关闭密码登录，仅保留密钥登录，以防被暴力破解。
