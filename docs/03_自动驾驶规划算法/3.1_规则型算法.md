# 3.1 规则型算法

传统的自动驾驶规划算法主要基于几何、物理规律和人为定义的逻辑规则。它们具有极高的可解释性和安全性，是目前绝大多数商用自动驾驶系统的核心。

## 1. 经典框架
- **任务分解**：路由 (Routing) -> 行为决策 (Behavior Decision) -> 运动规划 (Motion Planning)。
- **Frenet 坐标系**：将车辆在笛卡尔坐标系下的位置转化为相对于道路中心线的 $s$ (纵向) 和 $l$ (横向) 坐标，极大地简化了计算。

## 2. 经典算法
- **A* 与 Dijkstra**：用于全局路径搜索。
- **混合 A* (Hybrid A*)**：考虑了车辆的运动学约束（如最小转弯半径），适用于狭窄空间泊车或掉头。
- **Lattice Planning**：在 Frenet 空间进行采样，生成大量候选曲线，通过代价函数选出最优解。百度 Apollo 的规划方案深受此启发。
- **EM Planner**：通过 Expectation-Maximization 的迭代思路，交替优化路径和速度。
