# 4.1 传统端到端自动驾驶

“端到端” (End-to-End, E2E) 自动驾驶是指将感知、预测和规划整合到一个统一的模型中，直接从传感器输入（如摄像头图像、激光雷达点云）映射到驾驶动作（如轨迹点、控制信号）。这种范式旨在打破传统模块化管道中的“信息瓶颈”和“误差累积”。

## 1. 范式演进：从模块化到端到端

在传统的 **模块化架构 (Modular Pipeline)** 中，系统被切分为独立的模块：
`传感器 -> 感知 -> 预测 -> 决策规划 -> 控制`

这种架构虽然可解释性强，但存在以下痛点：
- **误差传播 (Error Propagation)**：感知层的一个微小错误（如漏检）会级联放大，导致最终规划失败。
- **目标不一致**：感知模块通常优化检测精度（mAP），而规划模块关注的是安全性，两者的优化目标往往不直接对齐。

### 端到端的优势与挑战
| 维度 | 传统模块化 | 端到端自动驾驶 |
| :--- | :--- | :--- |
| **优化目标** | 局部模块最优 | 全局任务最优（以规划为中心） |
| **计算效率** | 冗余计算多，中间表达复杂 | 特征共享，整体延迟通常更低 |
| **可解释性** | 极佳（每个模块输出明确） | 较弱（早期被称为“黑盒”） |
| **数据需求** | 需要大量带标注的感知数据 | 核心依赖专家轨迹数据 |

---

## 2. 核心数学表达：行为克隆 (Behavior Cloning)

大多数早期和现代端到端模型的核心是**行为克隆 (BC)**。给定专家驾驶数据集 $\mathcal{D} = \{(s_i, a_i)\}_{i=1}^N$，其中 $s$ 是环境状态，$a$ 是专家动作，模型 $\pi_\theta$ 的训练目标是：

$$\theta^* = \arg\min_\theta \sum_{(s, a) \in \mathcal{D}} \mathcal{L}(\pi_\theta(s), a)$$

其中 $\mathcal{L}$ 通常是均方误差 (MSE) 或交叉熵损失。

---

## 3. 现代里程碑：模块化端到端 (Modular E2E)

为了解决“黑盒”问题，现代研究倾向于在端到端网络中保留“模块化”的影子。这些模型通过加入中间监督任务（如语义分割、BEV 生成）来引导网络学习有意义的特征。

### 🌟 典型代表：UniAD (Unified Autonomous Driving)
UniAD (CVPR 2023 Best Paper) 是该领域的巅峰之作。它通过 **Query-based** 设计，将所有任务整合在 Transformer 架构中：

- **VQA (Vision Query Abstraction)**：提取统一的特征表达。
- **Track & Map Query**：同时处理跟踪和在线建图。
- **Motion & Occ Query**：预测周围障碍物的意图和占据网格。
- **Planning Query**：基于上述所有信息，输出安全的驾驶轨迹。

**UniAD 的核心贡献**：证明了多任务的协同学习不仅能提高可解释性，还能显著提升规划的安全性。

### 🌟 典型代表：VAD (Vectorized Autonomous Driving)
VAD 则追求极简的**矢量化**方案。它放弃了密集的占据网格（Occupancy），转而使用稀疏的矢量化表示（Vectorized representation）来建模环境，极大地降低了计算开销，提升了实时性。

## 4. 总结对比表

| 模型 | 发布年份 | 输入 | 输出 | 核心亮点 |
| :--- | :--- | :--- | :--- | :--- |
| **Dave-2** | 2016 | 单摄像头图像 | 方向盘转角 | 证明了 CNN 学习驾驶的可行性 |
| **UniAD** | 2023 | 六视角摄像头 | 未来驾驶轨迹 | 统一 Transformer 架构，多任务协同 |
| **VAD** | 2023 | 多视角摄像头 | 矢量化规划轨迹 | 极高性能，全矢量化表达 |
| **SparseAD**| 2024 | 环视图像 | 动作序列 | 进一步稀疏化，提升长时预测能力 |
